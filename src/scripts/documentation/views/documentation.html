Pitsby aims to be the easiest tool for generating a documentation website for your components, offering support for legacy and modern technologies. No rules. No loaders. No regex. Get a documentation website up and running with few super simple declarations.

This is Pitsby's configuration file. pitsby.config.js should be created in the root directory of the project which you want the documentation will be created for.

You should create one *.doc.js file per component per engine. That said, be careful about:

  • If you have a button component written in Vue and a button component written in React, both will need their own .doc.js.
  • The file below is just an example of how you write documentation files considering all supported technologies. On practice, you shouldn't ever mix technologies in the same *.doc.js.
